## 1. 프로젝트 목표

와인 병 사진을 올리면 AI가 다음 정보를 정리하는 모바일 웹앱을 만든다.

- 와인명
- 와이너리
- 빈티지
- 국가와 산지
- 포도 품종
- 와인 종류
- 테이스팅 요약
- 사람들의 코멘트 경향
- 재미있는 정보
- 추천 음식 페어링
- 사진에서 실제로 함께 먹은 음식
- AI 분석 신뢰도와 확인 상태
- 정보 확인에 사용한 출처

추천 페어링과 실제로 먹은 음식은 구분한다.

- `pairing`: 일반적으로 이 와인과 잘 어울리는 추천 음식
- `consumedFoods`: 사진에서 확인된, 이날 실제로 함께 먹은 음식
- `foodNote`: 음식 식별에 관한 설명이나 주의사항
- `foodConfidence`: 음식 식별 신뢰도

## 4. 아직 필요한 비밀정보

### OpenAI API 키

실제 AI 사진 분석을 활성화하려면 서버 환경변수 `OPENAI_API_KEY`가 필요하다.

- 현재 상태: 연결되지 않음
- 키가 없을 때: 서버가 `503`과 안내 메시지 반환
- 발급 위치: <https://platform.openai.com/api-keys>
- ChatGPT 구독과 OpenAI API 사용료는 별도
- API 키를 브라우저 JavaScript나 공개 파일에 직접 넣으면 안 됨
- API 키는 호스팅 서버의 비밀 환경변수로만 저장해야 함

현재 설정된 분석 모델:

```text
gpt-5.6-luna
```

서버 환경변수 `OPENAI_MODEL`에서 변경할 수 있다.

### 현재 구조적 제한

- 다른 휴대폰이나 컴퓨터에는 자동 동기화되지 않는다.
- 한 사람이 추가한 기록이 다른 공개 방문자에게 자동으로 나타나지 않는다.
- 이를 해결하려면 온라인 데이터베이스와 사용자 인증이 필요하다.

## 8. API 관련 개념

API는 프로그램끼리 요청과 결과를 주고받는 통로다.

이 프로젝트의 흐름:

```text
휴대폰에서 사진 선택
        ↓
웹앱이 서버 API에 사진 전달
        ↓
서버가 관리자 코드 확인
        ↓
서버가 비밀 API 키를 붙여 OpenAI에 요청
        ↓
AI가 와인과 음식 분석
        ↓
서버가 구조화된 결과 반환
        ↓
웹앱이 와인 카드로 표시
```

### API 키

OpenAI API를 사용할 권한이 있음을 증명하는 비밀 출입증이다.

- 외부에 공개하면 안 됨
- Git에 저장하면 안 됨
- 브라우저에 직접 넣으면 안 됨
- 서버 환경변수로 보관

### Responses API

텍스트와 이미지 등을 AI에 보내고 결과를 받는 OpenAI API다. 이 프로젝트에서는 와인 사진과 분석 지시문을 함께 전송하도록 구현했다.

### JSON

프로그램이 데이터를 주고받는 형식이다.

```json
{
  "name": "Mucho Más Tinto",
  "country": "스페인",
  "grapes": ["Tempranillo", "Garnacha", "Syrah"]
}
```

### JSON Schema

AI가 반환해야 하는 데이터의 형식을 미리 정의하는 규칙이다. 와인명, 국가, 음식 목록, 신뢰도 등이 빠지거나 잘못된 형태로 반환되는 것을 줄여준다.

## 10. Node.js와 npm

### Node.js

JavaScript를 브라우저 밖의 컴퓨터나 서버에서도 실행할 수 있게 하는 프로그램이다.

- React 코드 빌드

### npm

React, Vite 같은 JavaScript 부품을 설치하고 관리하는 도구다.

```text
npm install
npm run dev
npm run build
```

- `npm install`: 필요한 부품 설치
- `npm run dev`: 개발용 웹앱 실행
- `npm run build`: 배포용 결과물 생성

현재 처음 작업한 로컬 환경에는 Node/npm이 설치되어 있지 않아 원격 호스팅 서버에서 빌드 검증을 수행했다.

- 창이 다시 나오면 일반 계정 비밀번호를 넣지 말고 취소

## 13. 작업 중 발생했던 주요 오류

### Cloudflare 522

의미:

```text
브라우저 → Cloudflare: 정상
Cloudflare → 웹앱 서버: 응답 시간 초과
```

원인은 초기 서버 런타임 방식이 호스팅 환경과 맞지 않았기 때문이다. 서버 렌더링 의존성을 제거하고 안정적인 정적 웹앱 구조로 변경했다.

### 404 Not Found

호스팅 서버는 실행됐지만 정적 HTML·JavaScript 파일 연결을 찾지 못했다. 배포 결과물에 화면과 필요한 자원을 포함하는 방식으로 수정했다.

### JavaScript 외계어 출력

압축된 JavaScript가 실행되지 않고 화면의 일반 글자로 표시된 문제다.

원인은 JavaScript를 HTML 안에 넣는 과정에서 `$&`, `$`` 같은 문자가 문자열 치환용 특수기호로 잘못 처리된 것이었다. 문자열 치환을 함수 방식으로 변경해 코드 변형을 방지했다.

## 14. 개인정보와 비용 보호

- 공개 페이지에는 API 키나 관리자 코드를 포함하지 않는다.
- API 키와 관리자 코드는 서버 환경변수로 저장한다.
- 일반 방문자는 와인 목록만 볼 수 있다.
- AI 요청에는 올바른 관리자 코드가 필요하다.
- AI가 틀릴 수 있으므로 신뢰도와 확인 상태를 함께 표시한다.
- 불확실한 음식이나 와인 정보는 추측하지 않고 확인 필요로 표시한다.
